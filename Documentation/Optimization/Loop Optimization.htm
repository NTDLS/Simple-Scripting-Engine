Loop Optimization


Try not to re-declare variables.

Variable declaration is expensive and destruction of those variables is nearly as intensive. Since variables are destroyed once they fall out of scope, you should attempt to declare heavily used variables in a scope where they can be reused without being constantly re-declared. An example of this would be when looping or repeatedly calling a function that declares several variables.

Save values that are expensive to acquire.

Lengths of string can change and various properties or functions can have their return value modified by code within your loop. Therefore, it is required that the engine reevaluate loop conditions on completion of each iteration. If you know that your loop condition will not change, then you should save values which are expensive to acquire in variables - you can then use these variables in your loop condition. While this is not a requirement, it can greatly improve performance in key areas of your code.
